# Looping Ai 랜딩페이지 — 설계 문서 (Design Spec)

- 작성일: 2026-06-24
- 상태: 승인됨 (사용자 "추천대로" 승인, 2026-06-24)
- 다음 단계: 구현 계획 작성(writing-plans) → 구현

---

## 1. 한 줄 요약

AI 바이브코더 **"Looping Ai"** 의 서비스(수주) 랜딩페이지. 방문자가 "이 사람에게 웹사이트/랜딩페이지 제작을 의뢰하고 싶다"고 느끼고 **문의 폼**으로 전환하게 만드는 것이 유일한 목표. 페이지 자체의 퀄리티(3D·애니메이션·디자인)가 곧 실력 증명이자 포트폴리오다.

## 2. 목표 / 성공 기준

- **단일 전환 목표(CTA)**: 문의 폼 제출 (이름·이메일·프로젝트 유형·내용 수집)
- 페이지가 곧 포트폴리오 — 비주얼/인터랙션 퀄리티가 높을수록 영업력이 올라감
- 5초 안에 "이게 뭐고 나에게 왜 좋은지" 전달
- 빠른 로딩, 모바일 최적화, 단일 CTA, 일관된 브랜드, SEO 기본기

## 3. 타겟 / 메시지

- 타겟: 웹사이트·랜딩페이지·AI 웹·대시보드가 필요한 개인/소상공인/스타트업
- 핵심 메시지: *생각만 던지면, 며칠 안에 진짜 동작하는 웹사이트가 나온다.*
- 헤드라인(초안): **"당신의 아이디어, 며칠 만에 웹으로."**
- 보조문구(초안): "랜딩페이지 · AI 웹사이트 · 대시보드를 바이브코딩으로 빠르게."
- 브랜드 컨셉: "Looping" = 반복/루프 → 빠른 반복으로 아이디어를 굴려 완성한다는 정체성. 원형(루프) 그라데이션을 시각 모티프로 사용.

## 4. 디자인 시스템 (다크 프리미엄 + 네온)

- **배경**: 거의 검정 `#07070C`, 위에 움직이는 그라데이션 메시 블롭(보라→시안→마젠타)
- **포인트 컬러**: 일렉트릭 바이올렛(`#8B5CF6`계열) + 시안(`#22D3EE`계열) 그라데이션, 보조 마젠타
- **타이포**:
  - 헤드라인: 디스플레이 폰트 (Space Grotesk 또는 Clash Display)
  - 본문/한글: **Pretendard** (한국어 가독성 우선)
- **질감**: 글래스모피즘 카드, 미세 노이즈 그레인, 네온 글로우, 라운드 코너(2xl)
- 라이트/다크: 다크 단일 (라이트 토글은 비목표)

## 5. 섹션 구성 (위 → 아래)

1. **네비게이션** — 스티키 글래스 바. 로고(Looping Ai) / 작업물 · 서비스 · 후기 · 문의 / 「문의하기」 버튼
2. **히어로** — Spline 3D(아래 자산) 중앙 배경 + 헤드라인 + 보조문구 + CTA + 스크롤 큐. 등장 애니메이션(Framer Motion)
3. **실적 바** — 카운트업 숫자 (제작 건수 · 만족도 · 평균 제작기간 등). *실제 숫자는 사용자 제공 예정, 우선 예시값으로 채우고 교체 가능하게*
4. **서비스** — 3카드: 랜딩페이지 / AI 웹사이트 / 대시보드·웹앱. 호버 시 3D 틸트
5. **작업물(포트폴리오)** — 대표 케이스: World Cup Korea Dashboard (`https://worldcup-korea-dashboard.vercel.app/`). 카드 호버 미리보기 + 링크. 추가 작업물 슬롯 마련(데이터 배열로 관리)
6. **프로세스** — 1) 아이디어 공유 → 2) 빠른 프로토타입 → 3) 완성·배포. 스크롤 연동 타임라인
7. **차별점(Why Looping Ai)** — 빠름 · 합리적 가격 · 고퀄 · 직접 소통
8. **후기** — 고객 후기 카드 그리드/캐러셀 (*실제 후기 사용자 제공 예정, 예시 후기로 채움*)
9. **문의 폼 + 최종 CTA** — 이름 · 이메일 · 프로젝트 유형(select) · 내용(textarea) · 제출. 성공/실패 상태 표시
10. **푸터** — 브랜드 · 섹션 링크 · SNS · 카피라이트

## 6. 인터랙션 / 애니메이션 (전문성 포인트)

- 스크롤 리빌 + 스태거 (Framer Motion `whileInView`)
- 카드 3D 틸트(호버), **마그네틱 버튼**
- 움직이는 그라데이션 블롭 배경
- 숫자 카운트업 애니메이션
- 부드러운 스크롤 (Lenis)
- **Spline 3D 히어로** (핵심 자산)
- 기술/작업 마퀴(흐르는 띠)
- 성능 고려: 모바일에서 3D/무거운 효과는 조건부 경량화, `prefers-reduced-motion` 존중

## 7. 기술 구성

- **Next.js (App Router) + TypeScript**
- **Tailwind CSS** (디자인 토큰: 컬러/폰트/라운드/그림자)
- **Framer Motion** (애니메이션)
- **Spline**: `@splinetool/react-spline` 또는 iframe 임베드
  - 제공된 퍼블릭 URL: `https://my.spline.design/distortingtypography-HVtSRPvAe6lsZmpevIW7Ydk0/`
- 폰트: Pretendard(한글) + Space Grotesk/Clash(디스플레이)
- **부드러운 스크롤**: Lenis (선택)
- **배포**: Vercel

### 문의 폼 백엔드 (결정: Web3Forms)

- 기본: **Web3Forms** — 서버 비밀키 불필요, 공개 access key 하나로 동작. 정적/Vercel에서 즉시 작동.
  - 환경변수: `NEXT_PUBLIC_WEB3FORMS_KEY` (web3forms.com에서 무료 발급)
- 업그레이드 경로(선택): Next.js Route Handler(`app/api/contact/route.ts`) + **Resend** 이메일 발송. 키만 넣으면 전환 가능하도록 폼 제출 로직을 분리 설계.
- 클라이언트 검증(필수 필드, 이메일 형식) + 허니팟(스팸 방지) + 제출 중/성공/실패 상태 UI

## 8. 컴포넌트 경계 (구현 단위)

각 섹션을 독립 컴포넌트로 분리하고, 콘텐츠는 데이터 파일(`/data` 또는 상수)로 빼서 텍스트/숫자/포트폴리오를 코드 수정 없이 교체 가능하게 한다.

- `Nav`, `Hero`(+`SplineEmbed`), `StatsBar`, `Services`, `Portfolio`, `Process`, `WhyUs`, `Testimonials`, `ContactForm`, `Footer`
- 공용: `Section` 래퍼, `MagneticButton`, `TiltCard`, `GradientBlob`, `Counter`, `Marquee`
- 데이터: `services`, `portfolio`, `testimonials`, `stats`, `processSteps`

## 9. 콘텐츠 상태

- 사용자 제공: 포트폴리오 작업물(현재 1개 확정 = 월드컵 대시보드), 후기/실적 숫자(추후 실제값 전달)
- 미보유분은 **예시값/플레이스홀더로 채우고 교체 가능하게** 설계 (데이터 파일에서 한 곳만 수정)

## 10. 비목표 (YAGNI)

- 라이트 모드 토글, 다국어(영어) — 1차 범위 외 (한국어 단일). 영어는 차후 i18n로 확장 가능하게만 열어둠.
- 블로그/CMS, 결제, 회원가입/로그인 — 범위 외
- 복잡한 백엔드 — Web3Forms로 충분

## 11. 열린 항목 (구현 중 확정)

- 실제 실적 숫자/후기 텍스트 (사용자 제공 시 교체)
- 디스플레이 폰트 최종 선택 (Space Grotesk vs Clash Display)
- Spline은 react-spline 패키지 vs iframe — 로딩 성능 보고 결정
